# Standard DB Commands

This document defines the shared command contract for ORM and Alembic operations.
All FastAPI microservices should support the same command names and flags.

## Goals
- Keep developer workflow consistent across all services.
- Make CI scripts reusable.
- Reduce onboarding time for new teams.

## Required Commands

### 1) Create revision

```bash
make db-revision message="add campaigns table"
```

Expected behavior:
- Runs Alembic autogenerate.
- Creates a new file in `alembic/versions/`.
- Fails fast if `message` is missing.

### 2) Upgrade to head

```bash
make db-upgrade
```

Expected behavior:
- Applies all pending migrations to latest revision.

### 3) Downgrade

```bash
make db-downgrade step=-1
```

Expected behavior:
- Downgrades by provided revision step.
- Default `step=-1` if not provided.

### 4) Show current revision

```bash
make db-current
```

Expected behavior:
- Prints current migration revision(s) for active DB.

### 5) Show migration history

```bash
make db-history
```

Expected behavior:
- Prints full migration chain.

### 6) Drift check

```bash
make db-drift-check
```

Expected behavior:
- Runs autogenerate in check mode.
- Fails CI if model changes are not represented by migrations.

## Recommended Makefile Template

```make
ALEMBIC ?= alembic
STEP ?= -1

.PHONY: db-revision db-upgrade db-downgrade db-current db-history db-drift-check

db-revision:
	@if [ -z "$(message)" ]; then echo "Usage: make db-revision message=\"your message\""; exit 1; fi
	$(ALEMBIC) revision --autogenerate -m "$(message)"

db-upgrade:
	$(ALEMBIC) upgrade head

db-downgrade:
	$(ALEMBIC) downgrade $(STEP)

db-current:
	$(ALEMBIC) current

db-history:
	$(ALEMBIC) history

db-drift-check:
	$(ALEMBIC) check
```

## CI Command Sequence (Minimum)

```bash
make db-upgrade
make db-downgrade step=-1
make db-upgrade
make db-drift-check
```

## Team Rules for Command Usage
- Always run `make db-upgrade` locally before opening a PR.
- Run `make db-drift-check` after model changes.
- Do not merge schema changes without migration files.
- Review autogenerated migration files manually before commit.

## Notes
- If your Alembic version does not support `alembic check`, implement an equivalent script that fails when autogenerate detects pending changes.
- Keep command names stable so platform docs and CI pipelines remain reusable.
